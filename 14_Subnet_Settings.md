# 14 Subnet Settings (網段設定)

## 1. 功能目的

本功能用於管理系統中的網路資源（Subnet）。管理員可以對同步回來的網段進行分類（標註類型）、補充缺失的 CIDR/Gateway 資訊，並定義網段與資源池（Pool）之間的關聯。

正確的網段設定是虛擬機（VM）能夠成功分配 IP 並建立連線的基礎。

## 2. 使用者與權限

| 角色 | 操作權限 | 說明 |
|---|---|---|
| admin | 完整管理 | 可編輯網段屬性、手動補充 CIDR、綁定 Pool |
| viewer | 僅檢視 | 僅可檢視網段清單與詳細設定 |

## 3. 畫面與 Event 規格

### 3.1 網段管理清單

```text
+-----------------------------------------------------------------------+
| 網段管理                                                 [+ 手動新增] |
+-----------------------------------------------------------------------+
| 篩選：[雲端類型 ▼] [類型 ▼] [狀態 ▼]                 [搜尋網段名稱... Q] |
+-----------------------------------------------------------------------+
| 名稱/ID               CIDR             類型        關聯 Pool      狀態 |
+-----------------------------------------------------------------------+
| APP-Net               10.1.0.0/24     Public      Pool-A, Pool-B [ON ] |
| urn:vcloud:net:123..                                          [編輯] |
+-----------------------------------------------------------------------+
| MGMT-VLAN             192.168.1.0/24  Management  Pool-A         [OFF] |
| vlan-100                                                      [編輯] |
+-----------------------------------------------------------------------+
```

**畫面資料**

| 資料 | 用途 | 來源 |
|---|---|---|
| 網段清單 | 顯示 Subnet 名稱、CIDR 與用途分類 | `subnets` collection |
| 關聯 Pool 摘要 | 顯示哪些資源池可以使用此網段 | `subnets.pool_ids` 關聯之 `pools.name` |
| 狀態 | 標示網段是否可用於 VM 交付 | `subnets.status` |

**Event**

| UI 元件 | Event | 業務動作 | 異動/查詢資料 | 是否需要 Provider | 畫面結果 |
|---|---|---|---|---|---|
| 編輯按鈕 | 點擊 | 進入網段詳細設定 | 查詢 `subnets` 明細 | 否 | 跳轉至 3.2 編輯頁面 |
| 狀態切換 | 點擊 | 啟用或停用網段 | 異動 `subnets.status` | 否 | 狀態更新 |

```text
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| [資源池關聯]                                                           |
| 勾選可使用此網段的 Pool：                                               |
| [X] Pool-A (VDC-PROD)                                                 |
```

**Event**

| UI 元件 | Event | 業務動作 | 異動/查詢資料 | 是否需要 Provider | 畫面結果 |
|---|---|---|---|---|---|
| 儲存按鈕 | 點擊 | 更新網段設定與 Pool 關聯 | 更新 `subnets` 與關聯的 `pools.subnet_ids` | 否 | 回到清單頁 |

## 4. 主要業務資料異動流程

### 4.1 網段資訊補完流程
1.  **同步限制**：部分 Provider (如 vSphere) 可能無法自動帶出 CIDR 或 Gateway。
2.  **人工補件**：Admin 需在此頁面手動填入正確的網路資訊。
3.  **類型分類**：設定網段類型，這將影響 Apply Wizard 申請時的過濾邏輯（例如前端可能預設隱藏 Management 網路）。
