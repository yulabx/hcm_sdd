# 11 Project Dimension (專案維度檢視)

## 1. 功能目的

本功能提供以「專案」為核心的視視。不同於管理員關心的基礎設施容量，本頁面讓專案經理（Project Manager）檢視其專案下所有系統的資源佔用情況、已分配額度及 VM 清單。

## 2. 使用者與權限

| 角色 | 操作權限 | 說明 |
|---|---|---|
| admin | 全部專案檢視 | 可切換查看任何專案 |
| project_manager | 授權專案檢視 | 僅能看見其所屬專案的資料 |

## 3. 畫面與 Event 規格

### 3.1 專案資源摘要

```text
+-----------------------------------------------------------------------+
| 專案維度檢視：[ 專案 A (PRJ-ERP) ▼ ]                   [已申請] [使用量] |
+-----------------------------------------------------------------------+
| 資源配額使用情況：                                                       |
| CPU: [||||||||--] 80% (32/40 Cores)   Mem: [|||||-----] 50% (64/128 GB) |
+-----------------------------------------------------------------------+
| 系統清單：                                                              |
| 系統名稱      環境      VM 數    CPU/Mem/Disk      狀態                  |
| ERP-Web     Prod      3       12C / 24G / 300G  [健康]                |
| ERP-DB      Prod      2       16C / 32G / 1TB   [健康]                |
+-----------------------------------------------------------------------+
```

**畫面資料**

| 資料 | 用途 | 來源 |
|---|---|---|
| 配額使用率 | 顯示專案已領取的 Allocation 佔比 | `allocations` 加總 vs `projects` 預算 (若有) |
| Pool 容量視角 | Cloud / Site / Pool 卡片可切換已申請或使用量 | `pools.*_provisioned_*` 或 `pools.*_used_*` vs `pools.*_total_*` |
| 系統 VM 摘要 | 統計各系統下的虛擬機資源 | `vms` 依 `tags.system_id` 分群加總 |

**Event**

| UI 元件 | Event | 業務動作 | 異動/查詢資料 | 畫面結果 |
|---|---|---|---|---|
| 系統名稱連結 | 點擊 | 查看該系統的 VM 詳細清單 | 篩選 `vms` | 跳轉至 VM 管理頁並自動帶入篩選 |
| 資源視角切換 | 點擊 | 切換專案頁 Cloud / Site / Pool 容量圖表分子 | 讀取 `pools` 中 provisioned 或 used 欄位 | 容量百分比與 bar 即時更新 |

### 3.2 Pool 容量視角

專案維度頁面的容量圖表與資源總覽共用同一套資源視角。

| 模式 | 分子 | 分母 | 用途 |
|---|---|---|---|
| 已申請 | `*_provisioned_*` | `*_total_*` | 查看專案相關 Pool 的已分配、已申請或已承諾容量 |
| 使用量 | `*_used_*` | `*_total_*` | 查看專案相關 Pool 的實際消耗量 |

若已申請視角超過 100%，代表該 Pool 存在 overcommit 或 provider 以工作負載承諾量回報；UI 顯示真實百分比，bar 視覺寬度最多 100%。
