# 13 Pool Settings (資源池設定)

## 1. 功能目的

本功能提供管理員針對同步回來的資源池（Pool）進行人工介入管理。管理員可以修改 Pool 的顯示名稱、切換啟用/停用狀態、設定資源分配模式（Shared/Dedicated），以及最重要的——將 Pool 與可用的網段（Subnet）進行綁定。

資源池是 HCM 資源交付的核心，所有的資源申請（Apply Wizard）與資源核配（Allocation）都必須基於一個已啟用的資源池。

## 2. 使用者與權限

| 角色 | 操作權限 | 說明 |
|---|---|---|
| admin | 完整管理 | 可檢視、修改、切換狀態與綁定網段 |
| viewer | 僅檢視 | 僅可檢視資源池清單與設定細節 |
| project 類角色 | 無權限 | 此功能不對專案使用者開放 |

## 3. 畫面與 Event 規格

### 3.1 資源池管理清單

顯示系統中所有已同步或手動建立的資源池，並提供快速切換狀態的功能。

```text
+-----------------------------------------------------------------------+
| 資源池管理                                               [+ 新增資源池] |
+-----------------------------------------------------------------------+
| 篩選：[雲端類型 ▼] [狀態 ▼] [站點 ▼]                 [搜尋資源池名稱... Q] |
+-----------------------------------------------------------------------+
| 名稱/ID               雲端/站點      模式      容量摘要         狀態      操作 |
+-----------------------------------------------------------------------+
| VDC-PROD-01           VCD / Site-A  Shared   C:100/M:512/D:10  [ON ]    [編輯] |
| urn:vcloud:vdc:123...                                         [開關]    [刪除] |
+-----------------------------------------------------------------------+
| Cluster-H01           Harvester     Dedicated C:64/M:256/D:5   [OFF]    [編輯] |
| harvester-cluster-01  Site-B                                  [開關]    [刪除] |
+-----------------------------------------------------------------------+
```

**畫面資料**

| 資料 | 用途 | 來源 |
|---|---|---|
| 資源池清單 | 顯示 Pool 基本資訊、模式與容量摘要 | `pools` collection |
| 狀態 (active/inactive) | 以開關 (Toggle) 顯示目前是否可用於申請 | `pools.status` |
| 容量摘要 | 顯示 CPU (Core)、Mem (GB)、Disk (TB) 總量 | `pools.cpu_total`, `pools.mem_total_bytes`, `pools.disk_total_bytes` |
| 雲端與站點 | 標示資源池所屬的 Provider 與 Site | `pools.cloud`, `pools.site` |

**Event**

| UI 元件 | Event | 業務動作 | 異動/查詢資料 | 是否需要 Provider | 畫面結果 |
|---|---|---|---|---|---|
| 狀態開關 (Toggle) | 點擊 | 切換 Pool 狀態 | 異動 `pools.status` | 否 | 開關狀態更換，UI 提示「狀態已更新」 |
| 編輯按鈕 | 點擊 | 進入資源池編輯頁面 | 查詢該 Pool 明細 | 否 | 跳轉至 3.2 編輯頁面 |
| 新增資源池按鈕 | 點擊 | 手動建立非同步來源的 Pool | 建立 `pools` | 否 | 開啟新增表單彈窗 |
| 刪除按鈕 | 點擊 | 移除資源池記錄 | 刪除 `pools` | 否 | 二次確認後移除，清單重新整理 |

### 3.2 資源池編輯頁面

管理員修改 Pool 詳細設定與網段綁定的主畫面。

```text
+-----------------------------------------------------------------------+
| 編輯資源池：VDC-PROD-01                                     [儲存] [取消] |
+-----------------------------------------------------------------------+
| [基本設定]                                                             |
| 顯示名稱：[ VDC-PROD-01           ]  (原始名稱: vdc-01)                 |
| 資源模式：( ) Shared  ( ) Dedicated                                   |
| 所屬站點：[ Site-A        ▼ ]        狀態：( ) 啟用  ( ) 停用            |
| 適用環境：[X] Prod  [X] UAT  [ ] SIT  [ ] DR                          |
+-----------------------------------------------------------------------+
| [資源容量 (HCM 標準單位)]                                               |
| CPU 總量：[ 100 ] Cores              Mem 總量：[ 512 ] GB              |
| Disk 總量：[ 10  ] TB                                                 |
+-----------------------------------------------------------------------+
| [網段綁定 (Subnet Assignment)]                                         |
| [X] APP-Network (10.1.0.0/24) - Public                                |
| [X] DB-Network (10.2.0.0/24)  - Private                               |
| [ ] MGMT-Network (192.168.1.0/24) - Management                        |
+-----------------------------------------------------------------------+
```

**畫面資料**

| 資料 | 用途 | 來源 |
|---|---|---|
| 顯示名稱 | 修改使用者在申請頁面看到的名稱 | `pools.name` |
| 資源模式 | 決定此 Pool 走 Quota 分配 (Shared) 或整顆指派 (Dedicated) | `pools.type` |
| 適用環境 | 限制此 Pool 僅能被選定環境的系統申請 | `pools.allowed_vm_envs` |
| 資源容量欄位 | 人工校正或設定 Pool 的總容量限制 | `pools.cpu_total` 等 |
| 網段清單 | 顯示該 Cloud 下所有可選 Subnet 並供勾選 | `subnets` collection (需過濾 `cloud`) |
| 已綁定網段 | 標示目前哪些 Subnet 已關聯至此 Pool | `pools.subnet_ids` |

**Event**

| UI 元件 | Event | 業務動作 | 異動/查詢資料 | 是否需要 Provider | 畫面結果 |
|---|---|---|---|---|---|
| 網段勾選框 | 勾選/取消 | 建立或解除 Subnet 與 Pool 的關聯 | 暫存 `subnet_ids` 修改 | 否 | 勾選狀態異動 |
| 儲存按鈕 | 點擊 | 保存修改後的 Pool 設定 | 更新 `pools` 資料 | 否 | 回到清單頁，UI 提示「儲存成功」 |

## 4. 主要業務資料異動流程

### 4.1 資源池初始化與校正流程
1.  **資料來源**：透過 `12_Cloud_Settings.md` 的同步流程產生。
2.  **狀態預設**：新同步的 Pool 預設狀態為 `inactive`，必須由 Admin 校正。
3.  **模式設定**：Admin 需判斷此 Pool 是提供給多個專案共用 (Shared)，或是單一專案獨佔 (Dedicated)。
4.  **容量核定**：同步回來的容量若因 Provider 計算邏輯有誤（如 MHz 轉 Core），Admin 可在此進行最後修正。

### 4.2 網段關聯流程
1.  **重要性**：若一個 Pool 未綁定任何 Subnet，使用者在建立 VM 時將無網路可用。
2.  **多對多關係**：一個 Pool 可綁定多個 Subnet；一個 Subnet 也可以被多個 Pool 使用（例如跨 Cluster 的大網段）。
3.  **生效時機**：儲存後，Apply Wizard 選擇該 Pool 時會自動顯示其關聯的 Subnet。

## 7. 後續資料流向

1.  **Apply Wizard (15)**：僅會顯示 `status='active'` 且 `allowed_vm_envs` 符合的 Pool 供使用者申請。
2.  **Allocation Management (16)**：Admin 在分配資源時，僅能從已綁定的 Subnet 中選取分配給專案。
3.  **VM Management (17)**：建立 VM 時的 Subnet 下拉選單受限於 Pool 的 `subnet_ids`。
4.  **Resource Overview (10)**：以此功能的 `cpu_total` 等欄位作為 Dashboard容量計算的基準值 (Denominator)。
