# 10 Resource Overview (資源總覽)

## 1. 功能目的

本功能為 HCM 的首頁儀表板，提供管理員全系統的容量統計與資源分布概況。透過圖表呈現已同步資源的總量、已配置量與實際使用量，協助管理員判斷是否需要擴充雲端資源。

## 2. 使用者與權限

| 角色 | 操作權限 | 說明 |
|---|---|---|
| admin | 全系統檢視 | 可看見所有 Cloud、Site、Pool 的容量統計 |
| viewer | 全系統檢視 | 同 admin |
| project 角色 | 無權限 | 此頁面通常不對一般專案使用者開放，專案視角請看 11 |

## 3. 畫面與 Event 規格

### 3.1 資源熱點統計 (Dashboard)

```text
+-----------------------------------------------------------------------+
| 資源總覽                                      [已申請] [使用量] [重整資料 ↻] |
+-----------------------------------------------------------------------+
| [ 全系統 CPU ]          [ 全系統 Memory ]        [ 全系統 Disk ]        |
|    75% 使用中              60% 使用中              40% 使用中           |
|  (450/600 Cores)        (1.2/2.0 TB)            (40/100 TB)           |
+-----------------------------------------------------------------------+
| 雲端分布概況：                                                          |
| Provider    Pool 數    VM 數     CPU (分子/總量)    Mem (分子/總量)       |
| Harvester   2          45       120 / 200         400 / 800 GB        |
| AWS         1          12       10 / -            40 / -              |
+-----------------------------------------------------------------------+
```

**畫面資料**

| 資料 | 用途 | 來源 |
|---|---|---|
| 全系統統計 | 顯示加總後的資源百分比 | `pools` collection 加總計算 |
| Provider 分布 | 依雲端類型拆分顯示 | `pools` 依 `cloud` 分群加總 |
| 資源視角切換 | 切換容量圖表分子語意：已申請或使用量 | 前端全域狀態，預設已申請 |
| 容量百分比 | 依目前資源視角呈現分子佔總量比率 | `pool_provisioned / pool_total` 或 `pool_used / pool_total` |

## 4. 主要業務資料異動流程

### 4.1 容量計算邏輯
1.  **分母 (Total)**：來自於 Pool 的 `*_total_*` 欄位，例如 `cpu_total_millicores`、`mem_total_bytes`、`disk_total_bytes`。
2.  **已申請視角 (Committed)**：分子使用 `*_provisioned_*`，用於查看已分配、已申請或已承諾出去的容量；此為預設視角。
3.  **使用量視角 (Used)**：分子使用 `*_used_*`，用於查看 provider 回報的實際消耗量。
4.  **視覺呈現**：百分比文字保留真實值；若已申請視角因 overcommit 超過 100%，bar 視覺寬度 capped 在 100%，並以警示色呈現。
5.  **公有雲特殊性**：AWS 等公有雲 VPC 通常不提供 Total 容量，畫面上會顯示為 `N/A` 或僅顯示可取得的使用量。
