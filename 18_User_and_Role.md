# 18 User and Role (使用者與角色)

## 1. 功能目的

定義系統的存取控制規則（RBAC）。確保管理員可以執行全系統設定，而專案使用者僅能操作與其業務相關的資源，保障多租戶環境下的資料安全性。

## 2. 使用者與權限

| 角色 | 說明 | 主要權限範圍 |
|---|---|---|
| admin | 系統管理員 | 雲端設定、同步、資源池管理、網段管理、核配資源、所有 VM |
| project_manager | 專案經理 | 提交申請、查看所屬專案資源、操作所屬專案 VM |
| viewer | 稽核/檢視員 | 唯讀查看全系統容量與連線狀態，不可異動資料 |

## 3. 畫面與 Event 規格

### 3.1 使用者與專案授權管理

```text
+-----------------------------------------------------------------------+
| 使用者管理                                               [+ 新增使用者] |
+-----------------------------------------------------------------------+
| 帳號          姓名      角色            授權專案            狀態        |
+-----------------------------------------------------------------------+
| admin         管理者    admin           (全部)             [啟用]      |
| pm_user_01    張小明    project_manager  專案 A, 專案 B      [啟用]      |
| audit_01      稽核員    viewer          (全部-唯讀)         [停用]      |
+-----------------------------------------------------------------------+
```

**畫面資料**

| 資料 | 用途 | 來源 |
|---|---|---|
| 使用者清單 | 管理帳號與角色關係 | `users` collection |
| 授權專案 | 定義 project_manager 可以看到哪些資料 | `users.allowed_projects` (Project ID 陣列) |

**Event**

| UI 元件 | Event | 業務動作 | 異動/查詢資料 | 畫面結果 |
|---|---|---|---|---|
| 授權編輯 | 點擊 | 調整使用者所屬專案 | 更新 `users.allowed_projects` | 彈窗編輯授權清單 |

## 4. 權限矩陣 (RBAC Matrix)

| 功能模組 | admin | project_manager | viewer |
|---|---|---|---|
| Cloud Settings (同步) | 讀/寫 | 無權限 | 僅讀 |
| Pool/Subnet Settings | 讀/寫 | 無權限 | 僅讀 |
| Apply Wizard (申請) | 讀/寫 | 僅限授權專案 | 無權限 |
| Allocation Management | 讀/寫 | 無權限 | 僅讀 |
| VM Management (操作) | 讀/寫 | 僅限授權專案 | 僅讀 |
